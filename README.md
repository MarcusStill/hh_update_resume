# HH.ru Resume Auto-Lifter 🚀

Автоматический скрипт на Python для периодического поднятия резюме соискателя на сайте hh.ru в обход закрытого официального API. Скрипт эмулирует действия реального пользователя через скрытый браузер, проверяет доступность кнопки обновления и рассчитывает время «умного сна» до следующей разблокировки.

## ✨ Особенности

- **Полная автоматизация** — работает автономно 24/7 в фоновом режиме.
- **Умный расчёт времени** — скрипт парсит точное время разблокировки с сайта (например, *«Можно сегодня в 22:36»*) и засыпает ровно до этой минуты, не совершая холостых запросов.
- **Обход ограничений API** — работает через реальный браузер (Playwright Chromium), что защищает аккаунт от блокировок. HH.ru закрыли доступ к api для физ. лиц с конца прошлого года.
- **Фоновый режим** — возможность сборки в скрытый `.exe` файл без консольного окна.
- **Безопасное хранение сессии** — авторизация по СМС/паролю требуется только один раз. Данные в `auth_state.json` **обфусцируются** (XOR + Base64) для защиты от случайного просмотра.
- **Гибкая проверка авторизации** - определяет успешный вход по нескольким признакам: URL (содержит /applicant или role=applicant), видимым data-qa селекторам (mainmenu_resumes, applicant-resumes), а также видимым текстам (Мои резюме, Отклики, Резюме и профиль, Ваша активность). Это делает скрипт устойчивым к частым изменениям вёрстки HH.
- **Поддержка региональных поддоменов** - работает с любыми региональными адресами (например, belgorod.hh.ru) и сохраняет URL для корректной валидации сессии.
- **Автоматическое восстановление сессии** — при повреждении или устаревании файла сессии скрипт удаляет его и запрашивает новый вход без ручного вмешательства.
- **Защита от двойного запуска** — используется PID-файл, предотвращающий одновременную работу нескольких экземпляров.
- **Оптимизированная проверка** - тяжёлая операция чтения текста страницы выполняется только раз в 20 секунд, что снижает нагрузку на браузер.
- **Ротация логов** — файл `log.txt` автоматически ограничивается 5 МБ с хранением 3 архивных копий.
- **Расширенная диагностика при сбоях** — при неудачной авторизации сохраняются:
  - скриншот (auth_failed.png);
  - HTML-дамп страницы с меткой времени (auth_failed_YYYYMMDD_HHMMSS.html);
  - логируются URL, заголовок, статус окна, список кук и текст страницы;
  - старые HTML-файлы автоматически удаляются (остаются только 5 последних).

## 🛠️ Системные требования и установка

Для работы исходного кода вам понадобятся **Python 3.8+** и менеджер пакетов `pip`.

1. **Клонируйте или создайте папку проекта** и перейдите в нее.
2. **Установите необходимые библиотеки**:
   ```bash
   pip install -r requirements.txt
   ```

Первый запуск автоматически скачает браузер Chromium (около 150 МБ) в папку pw-browsers рядом со скриптом.

## ⚙️ Настройка файлов конфигурации

Рядом со скриптом автоматизации должны находиться два файла.

### 1. `config.ini` (Создать вручную)
Создайте текстовый файл, назовите его `config.ini` и укажите в нем ссылку на ваше резюме:

```ini
[SETTINGS]
# Интервал между плановыми проверками (в секундах), когда кнопка недоступна
check_interval_seconds = 14500

# Резервный интервал на случай ошибок (падение сайта, смена дизайна)
fallback_interval_seconds = 1800

[RESUMES]
# Уникальное имя = полная ссылка на ваше резюме
resume_main = https://hh.ru/resume/...
# resume_other = https://hh.ru/resume/...   (можно добавить несколько)
```

### 2. `auth_state.json` (Генерируется автоматически)

Файл содержит обфусцированные данные сессии (куки, токены). Создаётся при первом успешном входе и используется в дальнейшем для автоматической авторизации.

## 🚀 Инструкция по первому запуску (Авторизация)

1. Запустите скрипт:
   ```bash
   python hh_updater.py
   ```
2. Если файл auth_state.json отсутствует, откроется окно браузера с сайтом hh.ru.
3. Войдите в свой аккаунт соискателя (введите логин, пароль, СМС-код).
4. После успешного входа скрипт автоматически сохранит сессию (в обфусцированном виде) и закроет браузер.
5. Далее скрипт продолжит работу в фоновом режиме (headless).

Важно: повторный запуск не потребует повторного ввода — сессия будет загружена автоматически.

## 📦 Компиляция в фоновый `.exe` файл

Чтобы скрипт работал незаметно в диспетчере задач Windows и не держал открытым черное окно консоли, его можно упаковать в один исполняемый файл.

1. Установите упаковщик **PyInstaller**:
   ```bash
   pip install pyinstaller
   ```
2. Скомпилируйте проект:
   ```bash
   pyinstaller --onefile hh_updater.py
   ```
3. Готовый исполняемый файл появится в папке dist.
4. Перенесите hh_updater.exe в отдельную папку вместе с config.ini. При первом запуске он самостоятельно создаст auth_state.json и скачает браузер.

## 🔄 Автозагрузка Windows (Запуск при включении ПК)

Чтобы бот автоматически запускался в фоне при старте компьютера:
1. Нажмите комбинацию клавиш `Win + R`.
2. Введите команду `shell:startup` и нажмите Enter. Откроется папка автозагрузки.
3. Создайте **ярлык** для вашего файла `script.exe` и перетащите этот ярлык в открывшуюся папку.
4. При каждом включении компьютера скрипт будет запускаться автоматически в фоне.

## 🧠 Дополнительные возможности

* Автоматическая установка браузера — если браузер не обнаружен, скрипт самостоятельно скачает его через Playwright.
* Интеллектуальное восстановление — при любой ошибке загрузки сессии (повреждённый файл, смена пароля, истечение кук) скрипт перезапускает процесс авторизации.
* Множественные резюме — вы можете добавить несколько ссылок в секцию [RESUMES] с разными именами.
* Ротация логов — лог-файл не разрастётся до гигантских размеров, автоматически архивируется при достижении 5 МБ.
* Автоматическая очистка диагностических файлов — старые скриншоты и HTML-страницы удаляются, остаются только последние 5.

## ❓ Часто задаваемые вопросы
**Вопрос:** Нужно ли каждый раз вводить пароль и СМС?
**Ответ:** Нет, только при первом запуске или при сбросе сессии (например, при смене пароля на сайте).

**Вопрос:** Как узнать, что скрипт работает?
**Ответ:** Проверьте наличие процесса hh_updater.exe в диспетчере задач или посмотрите файл log.txt — там будет запись о каждом действии.

**Вопрос:** Что делать, если hh.ru изменил дизайн и кнопка не находится?
**Ответ:** В логе появится предупреждение, и скрипт перейдёт в резервный режим ожидания. Также будут созданы файлы auth_failed.png и auth_failed_*.html, по которым можно проанализировать текущее состояние страницы и обновить селекторы в коде (константы AUTH_SELECTORS, TEXT_LOCATORS, AUTH_TEXT_SNIPPETS в начале файла).

**Вопрос:** Где найти диагностические файлы?
**Ответ:** Они сохраняются в той же папке, где находится скрипт (или .exe). Скриншот всегда называется auth_failed.png, HTML-файлы имеют метку времени (например, auth_failed_20260710_225654.html).
